# Mini LinkedIn — API Backend
 
A recruitment platform REST API built with Laravel. Connects candidates and recruiters. Candidates can create profiles, list skills, and apply to job offers. Recruiters can post offers and manage applications. Admins supervise the platform.
 
---
 
## Tech Stack
 
- **Framework:** Laravel 11
- **Authentication:** JWT (tymon/jwt-auth)
- **Database:** MySQL
- **Language:** PHP 8.2+
---
 
## Prerequisites
 
- PHP >= 8.2
- Composer
- MySQL
- [tymon/jwt-auth](https://github.com/tymondesigns/jwt-auth)
---
 
## Installation
 
### 1. Clone the repository
 
```bash
git clone https://github.com/khawlaaas/mini-linkdin.git
cd mini-linkdin
```
 
### 2. Install dependencies
 
```bash
composer install
```
 
### 3. Set up environment
 
```bash
cp .env.example .env
php artisan key:generate
```
 
Open `.env` and fill in your database credentials:
 
```
DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=mini_linkedin
DB_USERNAME=root
DB_PASSWORD=
```
 
### 4. Generate JWT secret
 
```bash
php artisan jwt:secret
```
 
### 5. Run migrations and seeders
 
```bash
php artisan migrate --seed
```
 
This creates:
- 2 admin accounts
- 5 recruiters with 2–3 job offers each
- 10 candidates with profiles and skills
### 6. Start the server
 
```bash
php artisan serve
```
 
API is available at `http://localhost:8000/api`

## Postman Collection
 
Import the collection from `postman/mini-linkedin.postman_collection.json`.
 
Create a Postman environment with this variable:
 
| Variable | Value |
|---|---|
| `base_url` | `http://localhost:8000/api` |
 
After logging in, copy the `access_token` from the response and set it as Bearer Token on each request, or save it to an environment variable.

## API Routes
 
### Authentication
 
| Method | Endpoint | Description | Auth |
|---|---|---|---|
| POST | `/api/register` | Register a new user | None |
| POST | `/api/login` | Login and get JWT token | None |
| POST | `/api/logout` | Logout | Bearer |
| GET | `/api/me` | Get current user | Bearer |
 
### Profile — Candidates only
 
| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/profil` | Create profile (once) |
| GET | `/api/profil` | View own profile |
| PUT | `/api/profil` | Update profile |
| POST | `/api/profil/competences` | Add skill with level |
| DELETE | `/api/profil/competences/{competence}` | Remove skill |
 
### Job Offers
 
| Method | Endpoint | Description | Auth |
|---|---|---|---|
| GET | `/api/offres` | List active offers (filter by localisation, type) | Bearer |
| GET | `/api/offres/{offre}` | Get offer detail | Bearer |
| POST | `/api/offres` | Create offer | Recruiter |
| PUT | `/api/offres/{offre}` | Update own offer | Recruiter (owner) |
| DELETE | `/api/offres/{offre}` | Delete own offer | Recruiter (owner) |
 
Supports pagination (10 per page), sorting by date, and filtering by `localisation` and `type`.
 
### Applications
 
| Method | Endpoint | Description | Auth |
|---|---|---|---|
| POST | `/api/offres/{offre}/candidater` | Apply to an offer | Candidate |
| GET | `/api/mes-candidatures` | List own applications | Candidate |
| GET | `/api/offres/{offre}/candidatures` | View applications for an offer | Recruiter (owner) |
| PATCH | `/api/candidatures/{candidature}/statut` | Change application status | Recruiter (owner) |
 
### Administration — Admin only
 
| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/admin/users` | List all users |
| DELETE | `/api/admin/users/{user}` | Delete a user account |
| PATCH | `/api/admin/offres/{offre}` | Activate / deactivate an offer |
 
---
 
## Role-Based Access
 
| Role | Access |
|---|---|
| `candidat` | Profile management, view offers, apply |
| `recruteur` | Post and manage own offers, view and process applications |
| `admin` | Full platform supervision |
 
Accessing another user's resource returns **403 Forbidden**.
Missing or invalid token returns **401 Unauthorized**.

## Events & Listeners
 
The app uses Laravel's Events & Listeners system to log activity to `storage/logs/candidatures.log`.
 
| Event | Trigger | Logged info |
|---|---|---|
| `CandidatureDeposee` | Candidate applies to an offer | Date, candidate name, offer title |
| `StatutCandidatureMis` | Recruiter changes application status | Date, old status, new status |

## Branch Strategy
 
Each feature was developed on a dedicated branch and merged into `main` via Pull Request:
 
- `feature/database` — Migrations, models, seeders, factories
- `feature/auth` — JWT authentication and role middleware
- `feature/admin` — Admin endpoints
- `feature/profiles` — Candidate profile and skills management
- `feature/offres` — Job offer CRUD with filters and pagination
- `feature/candidatures` — Application endpoints and ownership rules
- `feature/events-listeners` — Event & Listener logging system
- `feature/postman-collection` — Postman collection
